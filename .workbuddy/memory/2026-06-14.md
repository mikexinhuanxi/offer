# 2026-06-14 工作日志

## 简历评估报告 UI Bug 修复（3个问题）

### 问题1："通过"状态颜色不一致
- **根因**：`resumeAudit.ts` 的 `toAuditCheck()` 保留原始 `severity` 字段，前端 `App.tsx` 在渲染时对 passed=true 的项仍添加 `.error`/`.warning` CSS class。CSS 中 `.audit-status.error` 定义在 `.audit-status.passed` 之后，导致 severity=error 的通过项显示红色，severity=warning 的通过项显示绿色。
- **修复**：`App.tsx` 中两处渲染逻辑（audit-meter-segment 和 audit-status），改为 `check.passed ? 不加severity class : 加上severity class`

### 问题2：技能列表错误包含邮箱和用户名
- **根因**：`buildSkillCheck()` 用宽泛正则 `/[A-Za-z][A-Za-z0-9+#. -]{1,30}/` 匹配技术词汇，过滤条件仅要求包含字母，导致 `manxinhuanxi123`（用户名）和 `foxmail.com`（邮箱域名）被当作技能。
- **修复**：新增 `isNonSkillTerm()` 过滤函数 + `NON_SKILL_PATTERNS` 正则列表，过滤 email(@)、URL、域名、纯数字字母用户名等非技能内容。

### 问题3：JD 解读卡片布局 + 加分项出现"1""2"空序号
- **布局**：`.jd-interpretation-grid` 从固定2列 grid 改为 CSS `column-count: 2` 瀑布流布局（类似小红书），卡片高度自适应不再强制对齐。
- **加分项**：`tencentMatchBuilder.ts` 的 `splitText()` 新增过滤规则 `/^\d{1,2}[\.\、\s]*$/`，排除纯数字序号行（如 "1"、"2"、"1."）。

### 问题4：每个岗位都显示"5 个待补关键词"
- **根因**：`extractKeywords()` 正则 `[\u4e00-\u9fa5]{2,8}` 太宽泛（2字中文即命中），停用词仅 9 个 → 几乎每个岗位都能提取出 ≥5 个通用词。
- **修复**：
  - 中文关键词要求至少 3 个字（减少虚词干扰）
  - 停用词从 9 个扩展到 ~50 个（含能力/经验/熟悉/掌握/参与等通用描述词、城市名、学历学位等）
  - 英文术语和中文分开展开再合并，避免混合截断不均

### 问题5：筛选"校招"显示 0 个岗位
- **根因**：`buildFilteredShortlist()` 做硬过滤——腾讯返回的 5 个岗位 type 都含"实习"，选"校招"后全部被过滤掉，结果为空。
- **修复**：增加回退逻辑——当筛选结果为 0 时，自动回退显示全部岗位。宁可硬性条件待补多一点，也不要出现"暂无岗位"。
