# Offer 捕手方案
## 问题诊断
学生校招投递常见问题是岗位分散、JD 难理解、简历建议泛化。若只依赖本地岗位库或模型经验，会有岗位不真实、理由不可验证、评分/概率误导等风险。因此项目定位为：基于真实岗位，输出可解释、可执行的求职建议。
## 方案设计
系统采用前后端分离。前端为 Vite + React + TypeScript，提供简历上传、岗位推荐、简历评估、岗位定制、面试准备。后端为 Node.js + Express + TypeScript，提供简历提取、分析、岗位状态和刷新接口。

核心流程是：解析简历并生成画像；调用腾讯校招 WorkBuddy skill 获取 `join.qq.com` 真实岗位/JD；保留顺序、链接和匹配理由；再由大模型生成诊断、改写和面试建议。前端不展示分数、通过率、HC、薪酬或录取概率。
## AI 工具选型
模型选择阿里云百炼 OpenAI 兼容接口，默认 `qwen-plus`。其接口接近 Chat Completions，便于替换；中文简历理解和 JSON 输出适合本地 demo。岗位源选择腾讯校招 WorkBuddy skill，因为它能抓取官网岗位与 JD。模型负责理解和表达，岗位事实由官网约束。
## 关键配置
关键配置包括 `DASHSCOPE_API_KEY`、`DASHSCOPE_MODEL`、`PORT`、`JOB_SOURCE_PROVIDER`、`TENCENT_SKILL_DIR`、`TENCENT_SKILL_PYTHON`、`TENCENT_JD_CACHE_DB`。默认前端 `5173`，后端 `8787`。`auto` 调用腾讯 skill；`local` 使用本地岗位库。
## 迭代记录
项目已从“岗位匹配 demo”迭代为“腾讯校招辅导工作台”：接入 PDF/DOCX/TXT 简历解析和腾讯官网岗位；结果页改为多 Tab；表格升级为推荐卡片；新增 JD 五段解读、简历评估、岗位定制、面试准备、模拟面试和群面/HR 辅导。
## 效果评估
当前效果体现在三点：岗位推荐可追溯到官网链接；输出覆盖“推荐原因、缺口、改简历、备面试”闭环；测试覆盖岗位匹配、简历优化、简历评估、结果页渲染和筛选兜底。后续可补充真实投递反馈、转化率统计和 DOC/PDF 导出能力。
